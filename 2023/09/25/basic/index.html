图片设置

然后就是图片在hexo本地无法显示的解决方案，这里用的markdown编辑器是typora，且在复制图片时，事先需要在typora中设置图片存放在指定的文件夹（这里指博客的子目录：/source/ _posts）。

如果图片无法显示，原因之一是hexo将.md文件转为.html时，只识别绝对地址。这可以通过下载插件解决npm install hexo-asset-img --save 。注意这里不是images 。参照以下第一个链接的评论区：

hexo博客中插入图片失败——解决思路及个人最终解决办法_hexo无法插入图片_金牛大王的博客-CSDN博客

主题配置以及升级

关于主题next：Hexo-Next 主题博客个性化配置超详细，超全面(两万字)-CSDN博客

若想升级该主题到最新版，先备份原来版本的next文件。考虑手动更新，需要进入Github的主页，搜索关键词hexo next，找到最新的更新包，下载并解压到本地，然后拖动到hexo的子目录/source/themes（由于解压后的文件名带有版本号，最好重命名为next）。然后将旧版本和新版本的_config.yml用vscode打开，可以开启左右拆分，然后按照以往对主题文件的配置，通过命令Ctrl+H，确定关键词的位置，然后对照进行修改。

Waline评论系统的引入

在配置之前，需保证hexo和next都是新版的，next的操作参考上一节，而hexo的更新参考以下：

Hexo 博客和 NexT 主题版本升级 | 笑话人生 (cylong.com)

评论系统配置参考：

Hexo博客进阶：为 Next 主题添加 Waline 评论系统-CSDN博客

当配置好，需要进行注册，获得管理员权限，这个需要本人登录到Login – Vercel，找到自己（按照上述链接）建立的project，然后点击右上角的visit，你会看到一个评论框；再点击评论框的注册（选择中文模式），会弹出一个小框，再点击其中的用户注册，值得一提的是，个人网站指的是自己博客的网址，而邮箱和密码是登录用的。若注册成功，再进行登录，进入后会发现左上角有一个管理按钮，这说明自己是管理员了。万一搞不清自己用哪个账号作为管理员，可以点击左上角的用户，来选择对应账号为管理员，可以有多个管理员。
