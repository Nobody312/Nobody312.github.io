hexo一些操作和配置

为了在将来某天换电脑时不会忘掉怎么配置，故记录以下所有的操作：

一开始需要下载各种软件，并进行配置；然后为了Github，需要得到id_rsa.pub（一般位于C盘）中的SSH内容，然后验证本地与Github仓库是否连接成功；接着对Github中的page进行设置（已经配置好了，选择main为默认，且Repository name已经修改成“用户名/github.io”，目的是为了让hexo本地文件可以更加顺利部署到Github仓库）。

2022 Hexo 博客搭建和使用教程(Windows) - 知乎 (zhihu.com)

上述提到的“部署”，具体操作为：基于Hexo和GitHub免费搭建个人博客网站 - 知乎 (zhihu.com)

hexo + github pages搭建博客样式加载不出来_banjw的博客-CSDN博客

值得注意的是，链接中提到的 _config.yml 在本机中可以用vscode打开并修改，若是新的机子，这个文件并没有一些选项，需要自己手动添加。且最好用SSH进行部署，若用https修改 _config.yml 配置，可能无法有效：

Hexo使用SSH连接GitHub_codepandaes的博客-CSDN博客

然后就是图片在hexo本地无法显示的解决方案，这里用的markdown编辑器是typora，且在复制图片时，事先需要在typora中设置图片存放在指定的文件夹（这里指博客的子目录：\source\ _posts）。

如果图片无法显示，原因之一是hexo将.md文件转为.html时，只识别绝对地址。这可以通过下载插件解决npm install hexo-asset-img --save 。注意这里不是images 。参照以下第一个链接的评论区：

hexo博客中插入图片失败——解决思路及个人最终解决办法_hexo无法插入图片_金牛大王的博客-CSDN博客

Hexo + Typora + 开发Hexo插件 解决图片路径不一致 | yiyun’s Blog (moeci.com)

而在hexo控制图片的大小，本人亲测在typora中的放缩代码无法在hexo中有效，故考虑一些特殊语法，代价是无法在typora中预览，不过利用文字描述可能有提示作用：

Hexo 自定义图片大小 | -炎屾- (xring.net)

至于在本地利用hexo进行博客写作，以及想删除本地和网路文件，参考：

使用hexo新建、编辑并预览文章 - 知乎 (zhihu.com)

[用Github Pages+Hexo搭建博客之(七)如何删除一篇已经发布的文章 #成功解决：同时删除掉.deploy_git文件夹_hexo删除文章_夏普通的博客-CSDN博客](https://blog.csdn.net/qq_34243930/article/details/109046120#:~:text=具体步骤如下 1 第一步，去文件夹 source%2F_posts 下删除你想要删除的文章 2 第二步，删除 .deploy_git,后，再执行 hexo g ， hexo g 即可。 发现文章删除成功✔)
